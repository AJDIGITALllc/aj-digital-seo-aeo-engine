# Page Blueprints System

**Exact page-level structures for premium, high-converting, AI-search-optimized websites.**

---

## Purpose

Define proven page recipes that remove unnecessary decision-making during builds. These blueprints give Claude Code and related agents a set of structural starting points for every common page type.

Use with:

- `/CLAUDE.md`
- `/skills/web_building/million_dollar_websites.md`
- `/skills/web_building/design_system.md`
- `/skills/web_building/claude_templates.md`
- `/skills/web_building/anti_gravity_prompts.md`

---

## Scope

**Included:** Page-level structure, section ordering, content priorities, CTA mapping, AEO patterns, and quality checklists for homepage, landing page, service page, case study, contact/inquiry page, and supporting page types.

**Not included:** Visual design tokens (see `design_system.md`), CRO frameworks (see `million_dollar_websites.md`), component-level build prompts (see `claude_templates.md`).

---

## Core Rule

Do not invent page structure from scratch unless the business model truly requires it.

Start with the closest blueprint in this file, then adapt.

# 1. Universal Page Principles

## 1.1 Every Page Must Do Five Things

1. Clarify what the page is about within the first screen
2. Match user intent
3. Create trust quickly
4. Guide the next action clearly
5. Be easy for humans and AI systems to parse

## 1.2 Every Page Must Satisfy These Standards

- One primary intent per page
- One primary CTA per page
- Clear heading hierarchy
- Strong information scent
- Scannable structure
- Mobile-first usability
- Fast-loading assets
- Direct-answer blocks where relevant
- Internal linking to adjacent pages
- No decorative sections without purpose

## 1.3 Every Page Should Answer These Questions

- What is this?
- Who is it for?
- Why should I care?
- Why should I trust you?
- What should I do next?

---

# 2. Page Selection Rules

## Use the Homepage blueprint when

- the page is the front door to the brand
- multiple audience types need orientation
- the site needs brand positioning + conversion

## Use the Landing Page blueprint when

- traffic comes from ads, email, socials, outreach, or a focused campaign
- the page supports one offer and one CTA
- distractions should be minimized

## Use the Service Page blueprint when

- the goal is to explain a service in depth
- the user is comparing providers
- the page needs SEO + sales utility

## Use the Case Study blueprint when

- trust and proof are required
- the buyer needs evidence before inquiry or purchase
- the site needs stronger bottom-funnel conversion support

## Use the Contact / Inquiry blueprint when

- the goal is form completion or booking
- the visitor is already warmed up
- the page must reduce friction and anxiety

# 3. Homepage Blueprint

## Goal

Orient the visitor, communicate positioning, establish trust, and route the visitor to the most important next step.

## Best For

- agency websites
- personal brands
- studios
- SaaS brands with multiple entry points
- hybrid service/product businesses

## Information Priority

1. Positioning
2. Offer clarity
3. Trust
4. Core services or pathways
5. Proof
6. CTA

## Section Order

### 1. Announcement Bar (optional)

Use only if there is one important message:

- launch
- event
- limited offer
- new case study
- lead magnet

Do not use for generic clutter.

### 2. Header / Navigation

Must include:

- logo
- minimal nav
- primary CTA
- mobile-ready structure

### 3. Hero

Must communicate:

- who you help
- what outcome you deliver
- what makes you different

Structure:

- H1 with clear result
- supporting subheadline
- primary CTA
- secondary CTA only if necessary
- visual or product mockup

Formula: Outcome + audience + differentiator

Example pattern:
`We build high-converting websites for service brands that need better leads, stronger positioning, and AI-search visibility.`

### 4. Trust Strip

Use:

- logos
- featured-in mentions
- client categories
- credibility stats

Keep visually quiet and immediate.

### 5. Problem / Opportunity Section

Show that you understand the visitor's situation.

Structure:

- section headline
- 3 to 5 pain points or missed opportunities
- short transition into your solution

### 6. Solution / Value Proposition Section

Present the offer clearly.

Structure:

- what you do
- how it works
- why it performs better than generic alternatives

### 7. Services or Pathways Section

For agencies or multi-offer brands.

Use cards or columns for:

- main service categories
- who each is for
- link to deeper page

### 8. Signature Benefits Section

Translate features into outcomes.

Good categories:

- better conversion
- better trust
- better speed
- better ranking
- better AI discoverability
- easier scaling

### 9. Proof Section

Use:

- case studies
- before/after
- testimonials
- metrics
- client wins

If possible, lead with numbers.

### 10. Process Section

Reduce uncertainty.

Structure:

- 3 to 5 steps
- simple labels
- plain language
- show a controlled process

### 11. Featured Case Study

One strong story often outperforms multiple weak ones.

Include:

- client context
- problem
- intervention
- measurable result

### 12. FAQ

Use only the most important objections.

This section should also support AEO:

- concise question
- direct answer first
- supporting bullets second

### 13. Final CTA

Re-state the value and direct the next action.

### 14. Footer

Must include:

- nav
- legal links
- contact path
- trust reinforcement

## Homepage Copy Rules

- Brand positioning first, details second
- Avoid vague claims like "innovative" or "cutting-edge"
- Use proof before polish where possible
- Keep the homepage broad enough to orient, but focused enough to convert

## Homepage SEO / AEO Notes

- Optimize for brand + primary category intent
- Add direct-answer summaries under key H2s
- Internally link to service pages, case studies, and contact page
- Use schema where appropriate

# 4. Landing Page Blueprint

## Goal

Drive one action with minimal distraction.

## Best For

- paid traffic
- lead magnets
- webinar registration
- sales campaigns
- offer validation
- product launch pages

## Information Priority

1. Offer clarity
2. Immediate relevance
3. Proof
4. Objection handling
5. CTA repetition

## Rules

- Remove full-site distractions when possible
- Use one audience, one offer, one CTA
- Keep message match tight with ad/source traffic
- Repeat CTA throughout the page

## Section Order

### 1. Hero

Structure:

- precise promise
- subheadline with mechanism or value
- CTA
- support proof near CTA

### 2. Proof Block

Use:

- star ratings
- logos
- testimonial snippet
- usage numbers
- trust badges

### 3. What You Get

Clarify deliverables.

### 4. Why It Matters

Translate deliverables into results.

### 5. Objection Handling

Address:

- time
- cost
- complexity
- skepticism
- fit

### 6. Process / What Happens Next

Show exactly what happens after clicking.

### 7. Expanded Social Proof

Testimonials or outcome-driven case proof.

### 8. FAQ

Answer objections tightly.

### 9. Final CTA

High-contrast, low-friction.

## Landing Page Copy Rules

- Specific beats clever
- Shorter sections than homepage
- Message match is critical
- Repetition is acceptable if it sharpens clarity

## Landing Page SEO / AEO Notes

Most landing pages are conversion-first, but still:

- use semantic structure
- maintain clear H1/H2 logic
- create answer-ready FAQ blocks
- avoid thin or deceptive copy

# 5. Service Page Blueprint

## Goal

Convert high-intent visitors looking for a specific service.

## Best For

- SEO pages
- agency service pages
- consulting offers
- production services
- local or niche service demand

## Information Priority

1. Service clarity
2. Fit
3. Differentiation
4. Proof
5. Process
6. CTA

## Core Principle

A service page must sell and rank at the same time.

## Section Order

### 1. Hero

State:

- service name
- who it is for
- result delivered

### 2. Direct Answer Block

Answer in 40 to 75 words:

- what the service is
- who it helps
- what outcome it creates

This supports both human clarity and AI retrieval.

### 3. Who This Is For

Define ideal fit explicitly.

### 4. Problems This Service Solves

List concrete pain points.

### 5. What's Included

Use a scoped deliverables section.

### 6. Outcomes / Benefits

Translate deliverables into business results.

### 7. Why Our Approach Is Different

This is the differentiation block.

Good differentiators:

- process rigor
- design quality
- speed
- AI-native workflow
- strategy depth
- category expertise

### 8. Process

3 to 6 steps.

### 9. Proof

Testimonials, results, case studies, examples.

### 10. Pricing Guidance or Engagement Model

Not always full pricing, but some buying guidance helps.

Examples:

- starting at
- custom scope
- typical project range
- retainers vs one-time

### 11. FAQ

Must cover buying objections and service mechanics.

### 12. CTA

Drive to:

- inquiry
- booking
- strategy call
- proposal request

## Service Page Copy Rules

- Use category language the market already understands
- Avoid over-branding the service if search demand matters
- Be concrete about scope
- Use examples and evidence

## Service Page SEO / AEO Notes

- Target one primary keyword cluster
- Support with related entities and subtopics
- Add internal links to adjacent services and case studies
- Use FAQ schema where appropriate
- Include direct answer blocks under key H2s

# 6. Case Study Blueprint

## Goal

Convert skepticism into trust using evidence.

## Best For

- agencies
- consultants
- studios
- SaaS proof pages
- premium service offers

## Information Priority

1. Context
2. Problem
3. Intervention
4. Result
5. Transferable trust

## Section Order

### 1. Hero Summary

Include:

- client type
- core challenge
- key result

### 2. Snapshot Metrics

Use 3 to 5 headline metrics.

### 3. Client Background

Give enough context to make the story meaningful.

### 4. Challenge

Define the actual business problem.

### 5. Strategy

Explain the thinking.

### 6. Execution

Show what was built, changed, or implemented.

### 7. Results

Use numbers first where possible.

### 8. Visual Evidence

Screens, assets, diagrams, or comparison blocks.

### 9. Client Testimonial

Best if specific and outcome-oriented.

### 10. Key Takeaways

What this proves about your process or capability.

### 11. CTA

Invite the reader into a similar engagement.

## Case Study Copy Rules

- Focus on transformation, not self-congratulation
- Keep the story tight
- Use specifics over adjectives
- Tie outcomes back to business value

## Case Study SEO / AEO Notes

- Target branded + service-adjacent intent
- Structure clearly so AI systems can extract the challenge, solution, and result
- Use scannable subheads and metrics blocks

# 7. Contact / Inquiry Page Blueprint

## Goal

Maximize qualified inquiries while minimizing friction.

## Best For

- agency contact pages
- quote requests
- discovery calls
- consultative sales

## Information Priority

1. What happens here
2. Who should inquire
3. What to expect
4. Reduced friction
5. Trust

## Section Order

### 1. Header + Intro

State:

- what the user is doing
- what happens after submission
- expected response timeline

### 2. Fit Guidance

Clarify:

- who this is for
- who it is not for
- project types or budget range if useful

### 3. Inquiry Form

Ask only what is needed.

Recommended fields:

- name
- email
- company
- website
- project type
- timeline
- budget range
- brief project summary

### 4. Alternate Contact Paths

Optional:

- email
- calendar booking
- social proof
- location

### 5. Trust / Assurance Block

Reduce submission hesitation.

Examples:

- no-pressure consultation
- response within X business days
- confidential inquiry
- strategy-first conversation

### 6. FAQ

Handle common hesitation.

## Contact Page Rules

- Shorter is usually better
- Reduce non-essential fields
- Make next steps explicit
- Reassure serious prospects

# 9. Blueprint Decision Matrix

| Page Type | Primary Intent | Best Traffic Source | CTA Style | Depth |
|---|---|---|---|---|
| Homepage | Orientation + conversion | Direct / brand / referral | Moderate | Medium |
| Landing Page | Single conversion | Paid / campaign / email | Aggressive | Short to medium |
| Service Page | Rank + convert | Search / referral | Strong | Medium to long |
| Case Study | Proof | Bottom-funnel / referral | Moderate | Medium |
| Contact Page | Inquiry completion | Internal site traffic | Direct | Short |

# 10. CTA Mapping Rules

## Use These CTA Types by Page

- Homepage: primary consultation or learn-more path
- Landing page: direct action only
- Service page: consultation, quote, or audit request
- Case study page: "get similar results" type CTA
- Contact page: form submit or booking

## CTA Rules

- Use one primary CTA phrase per page
- Keep verbs concrete
- Avoid weak CTA text like "Submit" or "Learn More" when stronger intent is possible

Better examples:

- Book My Strategy Call
- Get My Website Audit
- Start My Project
- See If We're a Fit

---

# 11. AEO Block Standard

For any page likely to be retrieved by AI systems, include this pattern under major H2 sections:

### Question-style H2

Example: `## What does a high-converting service page include?`

Then:

- 40 to 75 word direct answer
- 3 to 5 supporting bullets
- optional comparison or example

This pattern increases extraction clarity.

---

# 12. Section Quality Checklist

Before approving any page, confirm:

- [ ] The page has one clear primary intent
- [ ] The hero is specific and outcome-driven
- [ ] Trust appears early
- [ ] The page does not rely on generic adjectives
- [ ] CTA is repeated appropriately
- [ ] The structure matches actual user intent
- [ ] Headings are scannable
- [ ] Key answers are directly stated
- [ ] Mobile layout remains clean
- [ ] Every section earns its place

---

# 13. Common Failure Patterns

Do not approve pages with these issues:

- Hero says nothing specific
- Too many CTAs competing for attention
- Case studies without measurable outcomes
- Service pages that describe process but not result
- Homepages overloaded with equal-weight sections
- Contact pages with intimidating forms
- Landing pages with full website navigation and weak message match
- FAQ sections written as filler instead of objection handling
- "Pretty" pages with no conversion logic
